This twiki is a  list of the  E/gamma triggers run in 2016, 2017 and 2018.  It lists the run range they were active in, the menu version they first appeared in as well active and effective luminosities. <br>
<br>
Active lumi is the luminosity when the trigger prescale is not 0. <br>
Effective luminosity is the active luminosity times the HLT and L1 prescales.<br>
<br>
Due to paths being seeded by many L1 seeds, it is difficult to combine L1 prescales properly, so here the L1 prescale is simply the lowest prescale of its seeds that is non zero. There could be be accuracy issues here for edge cases so these luminosity values are simply a guide to use when picking your trigger and should not be used for detailed analysis. *It is advisable to use the brilcalc trigger path luminosity feature as a cross check*. Please let us know if you find an issue and we can correct it (this twiki is automatically generated so any edits here will be lost next time it is uploaded)<br>
<br>
Not all triggers are written out in physics streams, for example some (mostly for the tau group) triggers are run simply to record their decision but do not accept events. In this case the dataset is "NotFound".
<br>

This twiki also makes some simple recommendations that should be good for most analyses using electrons. If you have any questions about what trigger to use, please email us at hn-cms-egamma-hlt@cern.ch and we will be more than happy to check your choices. Its generally a good idea to do this at an early stage of the analysis because it is very easy to make a mistake here. <br>

Updates:<br>
   * 11/02/19: fixed bug in L1 prescales affecting lumiosity for all years (thanks to David Sperka for the spot!)

---++ Short Summary & Recommended Triggers:
These are simple generic recommendations for analyses. However, if you wish to maximise the sensitivity of your analysis, the use of prescaled triggers combined (or not) together is encouraged.

---++++Emulation of HLT_Ele32_WPTight_Gsf in 2017
An example on how you can emulate the HLT_Ele32_WPTight in 2017 is [[https://github.com/GhentAnalysis/heavyNeutrino/blob/0ac0591cea7a323356e5cc367c65213dbf1cfa44/multilep/src/TriggerAnalyzer.cc#L195-L215][here]] which is calling the function [[https://github.com/GhentAnalysis/heavyNeutrino/blob/0ac0591cea7a323356e5cc367c65213dbf1cfa44/multilep/src/TriggerAnalyzer.cc#L183-L193][here]]

This code is checking if there's an electron in the event which passes both the hltEle32L1DoubleEGWPTightGsfTrackIsoFilter (i.e. it is triggered by the HLT_Ele32_WPTight_Gsf_L1DoubleEG_v) and the hltEGL1SingleEGOrFilter filter (i.e. it would be triggered by a traditional HLT_Ele32_WPTight_Gsf if it was in the menu). In this way we can emulate the HLT_Ele32_WPTight_Gsf trigger.

(another example [[https://github.com/Sam-Harper/usercode/blob/106XNtup/TrigTools/plugins/Ele32DoubleL1ToSingleL1Example.cc][here]] does exactly the same)
